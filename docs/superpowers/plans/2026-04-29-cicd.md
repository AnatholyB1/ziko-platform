# CI/CD Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Set up two GitHub Actions workflows — quality gate + auto-deploy on push to `main`, and EAS build/submit on git tags.

**Architecture:** Two YAML workflow files in `.github/workflows/`. The `ci.yml` runs on every push to `main`: quality checks, then parallel Vercel deploy + Supabase migrations (conditionnel). The `release.yml` runs on tags `v*` or `beta*`: quality check, then EAS build + Play Store submit with different tracks depending on the tag prefix.

**Tech Stack:** GitHub Actions, Vercel CLI, Supabase CLI, EAS CLI (`expo/expo-github-action@v8`), Turborepo (existing `type-check` + `lint` scripts)

## Task 2: Créer `.github/workflows/ci.yml`

**Files:**
- Create: `.github/workflows/ci.yml`

Trois jobs : `quality` → puis `deploy-backend` et `migrate-supabase` en parallèle. Les migrations ne tournent que si des fichiers dans `supabase/migrations/` ont changé.

- [ ] **Step 1: Créer le dossier workflows**

```bash
mkdir -p .github/workflows
```

- [ ] **Step 2: Créer `.github/workflows/ci.yml`**

```yaml
name: CI

on:
  push:
    branches:
      - main

jobs:
  quality:
    name: Type-check & Lint
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'

      - name: Install dependencies
        run: npm ci

      - name: Type-check
        run: npm run type-check

      - name: Lint
        run: npm run lint

  deploy-backend:
    name: Deploy API to Vercel
    needs: quality
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Install Vercel CLI
        run: npm install -g vercel@latest

      - name: Deploy to production
        run: vercel --prod --yes --token=${{ secrets.VERCEL_TOKEN }}
        working-directory: backend/api
        env:
          VERCEL_ORG_ID: ${{ secrets.VERCEL_ORG_ID }}
          VERCEL_PROJECT_ID: ${{ secrets.VERCEL_PROJECT_ID }}

  migrate-supabase:
    name: Apply Supabase migrations
    needs: quality
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 2

      - name: Check for migration changes
        id: migrations
        run: |
          if git diff --name-only HEAD~1 HEAD | grep -q '^supabase/migrations/'; then
            echo "changed=true" >> $GITHUB_OUTPUT
          else
            echo "changed=false" >> $GITHUB_OUTPUT
          fi

      - name: Setup Supabase CLI
        if: steps.migrations.outputs.changed == 'true'
        uses: supabase/setup-cli@v1
        with:
          version: latest

      - name: Push migrations
        if: steps.migrations.outputs.changed == 'true'
        run: supabase db push --project-ref ${{ secrets.SUPABASE_PROJECT_ID }}
        env:
          SUPABASE_ACCESS_TOKEN: ${{ secrets.SUPABASE_ACCESS_TOKEN }}
```

## Task 3: Créer `.github/workflows/release.yml`

**Files:**
- Create: `.github/workflows/release.yml`

Déclenché sur les tags `v*` (production) et `beta*` (internal testing). Deux jobs : `quality` puis `eas-build-submit` avec branchement conditionnel selon le préfixe du tag.

- [ ] **Step 1: Créer `.github/workflows/release.yml`**

```yaml
name: Release

on:
  push:
    tags:
      - 'v*'
      - 'beta*'

jobs:
  quality:
    name: Type-check & Lint
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'

      - name: Install dependencies
        run: npm ci

      - name: Type-check
        run: npm run type-check

      - name: Lint
        run: npm run lint

  eas-build-submit:
    name: EAS Build & Submit
    needs: quality
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'

      - name: Setup EAS
        uses: expo/expo-github-action@v8
        with:
          eas-version: latest
          token: ${{ secrets.EXPO_TOKEN }}

      - name: Install dependencies
        run: npm ci

      # v* → build production (iOS + Android) + submit Android to production track
      - name: Build production (v* tag)
        if: startsWith(github.ref, 'refs/tags/v')
        run: eas build --platform all --profile production --non-interactive
        working-directory: apps/mobile

      - name: Submit Android to production track (v* tag)
        if: startsWith(github.ref, 'refs/tags/v')
        run: eas submit --platform android --profile production --non-interactive
        working-directory: apps/mobile

      # beta* → build preview Android + submit to internal track
      - name: Build preview Android (beta* tag)
        if: startsWith(github.ref, 'refs/tags/beta')
        run: eas build --platform android --profile preview --non-interactive
        working-directory: apps/mobile

      - name: Submit Android to internal track (beta* tag)
        if: startsWith(github.ref, 'refs/tags/beta')
        run: eas submit --platform android --profile internal --non-interactive
        working-directory: apps/mobile
```

- [ ] **Step 2: Commit**

```bash
rtk git add .github/workflows/release.yml
rtk git commit -m "ci: add release workflow — EAS build/submit on v* and beta* tags"
```

## Task 4: Configurer les secrets GitHub

**Files:** aucun fichier à modifier — configuration dans GitHub UI.

Ces étapes sont manuelles dans l'interface GitHub : **Settings → Secrets and variables → Actions → New repository secret**

- [ ] **Step 1: Obtenir `VERCEL_TOKEN`**

1. Aller sur vercel.com → Account Settings → Tokens
2. Créer un token avec scope "Full Account"
3. Ajouter comme secret GitHub : `VERCEL_TOKEN`

- [ ] **Step 2: Obtenir `VERCEL_ORG_ID` et `VERCEL_PROJECT_ID`**

Dans le terminal, depuis la racine du projet :
```bash
cd backend/api
vercel link
cat .vercel/project.json
```

Output exemple :
```json
{"orgId":"team_xxxx","projectId":"prj_yyyy"}
```

Ajouter :
- Secret `VERCEL_ORG_ID` = valeur de `orgId`
- Secret `VERCEL_PROJECT_ID` = valeur de `projectId`

Note : `.vercel/` est dans `.gitignore`, ne pas committer.

- [ ] **Step 3: Obtenir `SUPABASE_ACCESS_TOKEN` et `SUPABASE_PROJECT_ID`**

- `SUPABASE_ACCESS_TOKEN` : supabase.com → Account → Access Tokens → Generate new token
- `SUPABASE_PROJECT_ID` : Dashboard → ton projet → Settings → General → "Reference ID" (format : `abcdefghijklmnop`)

Ajouter les deux comme secrets GitHub.

- [ ] **Step 4: Obtenir `EXPO_TOKEN`**

1. Aller sur expo.dev → Account Settings → Access Tokens
2. Créer un token
3. Ajouter comme secret GitHub : `EXPO_TOKEN`
