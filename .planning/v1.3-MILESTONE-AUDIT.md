## Unsatisfied Requirements Detail

### ❌ RATE-01 — IP Flood Protection Not Active

**Expected:** `app.use('*', ipRateLimiter)` after CORS  
**Actual:** absent from app.ts  
**Effect:** Any unauthenticated IP can flood all non-AI endpoints indefinitely. The `/health`, `/plugins`, `/webhooks`, `/bugs`, `/supplements`, `/pantry` routes have no IP quota.  
**Fix:** Add import + `app.use('*', ipRateLimiter)` after CORS in app.ts

### ❌ SEC-01 — CORS Wildcard Regression

**Expected:** Only `exp://`, `localhost`, and explicit `APP_ORIGIN` in CORS allowlist  
**Actual:** `/^https?:\/\/.*\.vercel\.app$/` regex is back at line 25; `process.env.APP_ORIGIN ?? ''` allows empty-string bypass  
**Effect:** Any Vercel preview/deploy URL (attacker-controlled subdomain) can make authenticated cross-origin requests  
**Fix:** Remove the `*.vercel.app` regex line and the `?? ''` fallback from app.ts CORS config

### ❌ SEC-02 — No Security Headers

**Expected:** X-Frame-Options, X-Content-Type-Options, Referrer-Policy, Strict-Transport-Security on every response  
**Actual:** `secureHeaders()` not imported, not applied  
**Effect:** No clickjacking protection, MIME-sniffing protection, or HSTS enforcement  
**Fix:** Add `import { secureHeaders } from 'hono/secure-headers'` and `app.use('*', secureHeaders())` after CORS

### ⚠ SEC-03 — ZodError Handler Missing (partial)

**Expected:** `if (err instanceof z.ZodError) return c.json({ ... }, 400)` in `app.onError`  
**Actual:** `app.onError` only logs and returns 500; `import { z }` absent  
**Effect (limited):** Route-level `zValidator` still returns 400 for all three AI routes — the primary protection is intact. Only `z.ZodError` thrown inside handler bodies would return 500 instead of 400 (currently no handler does this, but it's a safety net regression).  
**Fix:** Restore `import { z } from 'zod'` and the ZodError branch in `app.onError`
