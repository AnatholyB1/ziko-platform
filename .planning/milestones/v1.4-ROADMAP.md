# Milestone v1.4: Système de Crédits IA & Monétisation

**Status:** ✅ SHIPPED 2026-04-29
**Phases:** 17–21
**Total Plans:** 11

## Overview

Implemented a gamified AI credit system that controls API costs (max €0.75/month per freemium user) while rewarding user engagement with activity-based credit earning. The milestone delivers a dual balance (shop coins + AI credits), atomic PostgreSQL deduction with SECURITY DEFINER concurrency safety, Haiku vision migration for ~70% per-scan cost reduction, fire-and-forget earn hooks on 6 activity types, and a complete mobile credit UI (balance chip, earn toast, exhaustion bottom sheet, cost labels, dual-balance card).

## Milestone Summary

**Key Decisions:**
- SECURITY DEFINER + SELECT FOR UPDATE in `deduct_ai_credits` — prevents negative balances under concurrent Vercel Fluid Compute requests (application-layer check-then-deduct races)
- Partial unique index `WHERE idempotency_key IS NOT NULL` on `ai_credit_transactions` — ON CONFLICT DO NOTHING eliminates double-crediting on mobile retry
- Lazy daily-reset (date-keyed check at earn time) — no cron dependency; avoids Vercel at-least-once cron delivery causing double-resets
- Redis (Upstash) is rate limiting only — credits live exclusively in Supabase for durable audit trail; 429 ≠ 402 (distinct concepts)
- `models.ts` as single model constants file — when model IDs change, update one file; prevents drift across 3+ backend consumers
- Separate `ai_credit_transactions` table (not gamification coins) — dual balance: coins are unlimited reward currency, credits are cost-controlled AI currency

**Issues Resolved:**
- Deprecated `claude-3-haiku-20240307` replaced with `claude-haiku-4-5-20251001` before April 19, 2026 retirement deadline
- AIBridge 200-char body truncation (CRED-05) — extended to 500 chars; exhaustion sheet now triggers correctly when `earned_today` has any source
- CRED-03 program quota dead code — `/ai/programs/generate` route created with proper `creditCheck('program')` gate
- Pre-existing `Alert.alert` violations in MeasurementsLog.tsx and GamificationDashboard.tsx replaced with `showAlert`

**Technical Debt Incurred:**
- Hardcoded model strings in `onFinish logTokenUsage` callbacks — if model IDs change in `config/models.ts`, telemetry strings drift (low-impact: telemetry only)
- 4 visual verification tests for Phase 21 require a running Expo app (balance chip, exhaustion sheet animation, earn toast animation, dual-balance card layout)
- Phases 17 and 18 VALIDATION.md in `status: draft` — no test runner in backend/api (tsc --noEmit is primary verification for SQL+service work)
