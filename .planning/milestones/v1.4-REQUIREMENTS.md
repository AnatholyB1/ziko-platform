# Requirements Archive: v1.4 — Système de Crédits IA & Monétisation

**Archived:** 2026-04-29
**Milestone shipped:** 2026-04-29
**Core Value:** A fitness user has a single app that coaches them, tracks everything, tells them what to cook based on what's in their kitchen — and controls AI costs through gamified engagement.

---

## v1.4 Requirements — Final Status

### Credit System

- [x] **CRED-01**: User can see their AI credit balance in the app at all times
  - *Outcome: balance chip in AI chat screen header + dual-balance card in gamification dashboard — Phase 21*
- [x] **CRED-02**: User receives a daily base allocation (1 photo scan + 1 AI chat) without any activity
  - *Outcome: DAILY_QUOTAS `{chat:{base:1}, scan:{base:1}}` enforced by creditGate middleware — Phase 18*
- [x] **CRED-03**: User receives 1 free AI program generation per month without any activity
  - *Outcome: `/ai/programs/generate` route created with `creditCheck('program')` + `creditDeduct('program')` — Phase 21 gap fix (2026-04-29)*
- [x] **CRED-04**: User sees the credit cost displayed next to each AI action button before using it
  - *Outcome: cost labels on send button (chat), camera/gallery buttons (scan), generate button (program) — Phase 21*
- [x] **CRED-05**: User sees an exhaustion bottom sheet when credits reach 0, explaining why and how to earn more
  - *Outcome: CreditExhaustionSheet with 6-activity checklist; AIBridge slice extended to 500 chars to parse 402 body — Phase 21 + gap fix (2026-04-29)*
- [x] **CRED-06**: User's credits are deducted atomically via PostgreSQL RPC (no negative balance possible)
  - *Outcome: `deduct_ai_credits` SECURITY DEFINER + SELECT FOR UPDATE + CHECK(balance>=0) — Phase 17*
- [x] **CRED-07**: User has a separate AI credits balance from shop coins (dual balance)
  - *Outcome: `user_ai_credits` table separate from gamification coins; dual-balance card in UI — Phase 17 + 21*

### Activity Earn

- [x] **EARN-01**: User earns AI credits by logging a workout
  - *Outcome: `workoutStore.endSession` → `callCreditsEarnWithResult(source:'workout')` — Phase 20*
- [x] **EARN-02**: User earns AI credits by completing daily habits
  - *Outcome: `HabitsDashboardScreen.handleToggle/handleIncrement` → earn hook — Phase 20*
- [x] **EARN-03**: User earns AI credits by logging meals
  - *Outcome: `LogMealScreen.saveLog` → earn hook — Phase 20*
- [x] **EARN-04**: User earns AI credits by logging body measurements
  - *Outcome: `MeasurementsLog.handleSave` → earn hook — Phase 20*
- [x] **EARN-05**: User earns AI credits by completing a stretching session
  - *Outcome: `StretchingSession.handleNext` → earn hook — Phase 20*
- [x] **EARN-06**: User earns AI credits by completing a cardio/running session
  - *Outcome: `CardioTracker.handleSave` → earn hook — Phase 20*
- [x] **EARN-07**: User's daily earned credits are capped (bonus max not exceeded)
  - *Outcome: DAILY_EARN_CAP=4 enforced in `creditService.earnCredits` — Phase 18*
- [x] **EARN-08**: User sees a "+1 AI credit" toast after earning credits from an activity
  - *Outcome: `CreditEarnToast` MotiView; `.then((r) => { if (r.credited) showEarnToast() })` on all 6 save handlers — Phase 21*
- [x] **EARN-09**: User sees daily earn progress ("2 credits available — log a stretch to earn")
  - *Outcome: daily earn progress bar in gamification dashboard; exhaustion sheet shows earned_today checklist — Phase 21*
- [x] **EARN-10**: Earned credits are idempotent (mobile retry does not double-credit)
  - *Outcome: partial unique index on `(user_id, source, idempotency_key) WHERE idempotency_key IS NOT NULL`; ON CONFLICT DO NOTHING — Phase 17 + 18*

### Vision & Cost

- [x] **COST-01**: Photo scan uses Claude Haiku 4.5 instead of Sonnet (~70% cost reduction)
  - *Outcome: VISION_MODEL=`claude-haiku-4-5-20251001` in `config/models.ts`; vision route upgraded — Phase 17 + 19*
- [x] **COST-02**: Each AI API call's token usage is logged to Supabase for cost tracking
  - *Outcome: `ai_cost_log` table (migration 027) + `logTokenUsage` in `onFinish` callback on all chat routes — Phase 19*
- [x] **COST-03**: Monthly cost per active freemium user stays under €0.75
  - *Outcome: cost ceiling verified via measured token counts and model pricing — Phase 19*

### Premium Prep

- [x] **PREM-01**: User profile has a tier column (free/premium) — premium users bypass credit gate
  - *Outcome: `user_profiles.tier TEXT NOT NULL DEFAULT 'free'` — migration 026, Phase 17*
- [x] **PREM-02**: Credit gate middleware checks tier before deducting credits
  - *Outcome: `creditGate.ts` queries `user_profiles.tier`; premium bypasses deduction — Phase 18*

## Out of Scope (unchanged)

| Feature | Reason |
|---------|--------|
| In-app purchases (IAP) | RevenueCat integration, 30% platform cut, RGPD compliance — future milestone |
| Credit rollover | Breaks cost projection — accumulated balances cause unpredictable API spikes |
| Referral/gift credits | Abuse vector complexity exceeds v1.4 scope |
| Credit usage history screen | Add when support tickets about balance appear |
| Streak multiplier | Requires dedicated streak table, timezone logic — defer |
| Dark mode credit UI | Light sport theme only — project constraint |
| Coins-to-credits conversion | Arbitrage anti-pattern destroys gamification economy |
