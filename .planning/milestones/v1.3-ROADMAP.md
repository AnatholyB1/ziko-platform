## Milestone Goal

Securiser le backend Hono contre les abus avec du rate limiting distribue via Upstash Redis, durcir l'API (CORS, headers, validation Zod), et gerer les assets media via Supabase Storage avec upload direct depuis le mobile et lifecycle policies. Phase 16 cloture les gaps de l'audit (regression middleware dans app.ts).

---

## Key Accomplishments

1. **Distributed Rate Limiting** — Upstash Redis HTTP client (serverless-safe) with sliding window algorithm; global IP flood protection (200 req/60s) + per-user quotas on AI chat (20/60min), tools (30/60min), and barcode scan (20/60min); all 429 responses include Retry-After header
2. **API Security Hardening** — CORS locked to explicit origins (no `*.vercel.app` wildcard), `secureHeaders()` on every response (X-Frame-Options, HSTS, etc.), Zod `.strict()` input validation on all AI POST routes via `@hono/zod-validator`, ZodError handler returning 400
3. **Supabase Storage** — Three private buckets (profile-photos, scan-photos, exports) with path-prefix RLS policies; signed URL upload flow bypasses Vercel's 4.5 MB body limit; mobile clients upload directly to Supabase; AI vision endpoint accepts `storage_path` for signed download URLs to Claude
4. **Lifecycle Cleanup** — Vercel cron runs daily at 4am UTC; purges scan-photos older than 90 days and exports older than 7 days via Supabase Storage JS client; CRON_SECRET authentication; fault-tolerant (Promise.allSettled per folder)
5. **Regression Fix** — Phase 15 commit accidentally reverted Phase 12+13 middleware; detected by milestone audit; Phase 16 restored all four items in a single targeted edit
