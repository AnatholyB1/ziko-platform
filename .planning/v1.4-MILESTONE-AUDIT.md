## Gaps

### Critical Gaps (blockers)

#### GAP-1: CRED-03 — Program Quota Dead Code

**REQ:** CRED-03 — User receives 1 free AI program generation per month without any activity
**Phase:** 18 (creditService.ts)
**Status:** ❌ unsatisfied

`getQuotaStatus('program')` and `MONTHLY_QUOTAS.program = { base: 1, bonus: 0 }` are implemented in `creditService.ts` but no route calls `creditCheck('program')` or `creditDeduct('program')`. AI program generation flows through `/ai/chat/stream` using the `'chat'` action — it costs a chat credit, not a program credit. The `'program'` action type in `CREDIT_COSTS` (cost: 4 credits) is defined but never deducted anywhere.

**Options:**
1. Create a dedicated `/ai/programs/generate` route with `creditCheck('program')` and `creditDeduct('program')` — builds the intended separate monthly quota gate
2. Rescope CRED-03 to "program generation costs a chat credit" — accept that program generation is already gated via the chat endpoint and retire the program-specific quota infrastructure

#### GAP-2: CRED-05 — AIBridge 200-char Truncation

**REQ:** CRED-05 — User sees exhaustion bottom sheet at 0 credits
**Phase:** 21 (AIScreen, creditStore)
**Status:** ⚠ partial — fails in the most common exhaustion case

**File:** `packages/ai-client/src/AIBridge.ts` line 201
**Bug:** `xhr.responseText.slice(0, 200)` — the 402 response body exceeds 200 chars when `earned_today` contains any source:
- `earned_today: []` = ~195 chars ✅ fits
- `earned_today: ["workout"]` = ~204 chars ❌ truncated, JSON.parse throws
- `earned_today: ["workout","habit","meal","measurement","stretch","cardio"]` = ~248 chars ❌ truncated

`AIScreen.handleSend` catches the thrown error, extracts the body substring via regex, calls `JSON.parse(match[1])` — throws on truncated input, inner `catch {}` swallows it silently, `showExhaustionSheet` never called.

**Fix:** `packages/ai-client/src/AIBridge.ts` line 201 — change `.slice(0, 200)` to `.slice(0, 500)` (or remove truncation entirely, the 402 body is known-bounded at ~250 chars max).

## E2E Flows

### Flow 1: User sends AI chat (sufficient credits) ✅ COMPLETE
`AIScreen.handleSend` → `POST /ai/chat/stream` → `creditCheck('chat')` (quota pass-through for base allocation) → handler → `creditDeduct('chat')` → deduction logged → response streams to mobile

### Flow 2: User logs workout → earns credit toast ✅ COMPLETE
`workoutStore.endSession` → `callCreditsEarnWithResult(supabase, 'workout', sessionId)` → `POST /credits/earn` → `creditService.earnCredits` → `earn_ai_credits` RPC → `{ credited: true }` → `useCreditStore.getState().showEarnToast()` → `CreditEarnToast` renders

### Flow 3: User hits 0 credits → exhaustion sheet ⚠ PARTIAL
`handleSend` → `POST /ai/chat/stream` → `creditCheck('chat')` (balance=0, quota exhausted) → 402 with body `{error, balance, earned_today, reset_timestamp}` → `AIBridge.ts` throws `"AI API error 402: ..."` with body sliced at 200 chars → `handleSend` catch → regex extract → `JSON.parse(body)` → **throws if earned_today ≥ 1 source** → `catch{}` swallows → exhaustion sheet never shown
