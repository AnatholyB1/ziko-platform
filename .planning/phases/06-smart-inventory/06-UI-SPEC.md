## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding |
| sm | 8px | Row item internal spacing, chip gaps |
| md | 16px | Default field padding, section internal spacing |
| lg | 20px | ScrollView horizontal padding (matches all plugins) |
| xl | 24px | Section heading margin-bottom |
| 2xl | 32px | Between major visual groups |
| tab | 100px | `paddingBottom` on every ScrollView â€” tab bar clearance |

Exceptions:
- Touch targets for icon-only buttons: minimum 44px (padding: 6 on a 24px icon = 36px hit, use padding: 10 to reach 44px)
- Camera modal overlay: full-screen, no padding on camera viewport
- Expiry dot indicator: 8px diameter, 4px margin-right

## Interaction Contracts

### Tap item row
Navigate to PantryItemForm pre-filled with existing item data (edit mode). Uses `router.push('/(app)/(plugins)/pantry/edit')` with item id param.

### Tap barcode button
Open camera modal. Auto-close on successful scan. Return to form with name pre-filled. Toast on not-found (remain on form, name field focused).

### Long-press item row
Not used. All destructive actions are in the edit form only.

### Delete item
Accessible only from edit form header ("Supprimer cet article"). Triggers `showAlert` with two-button confirmation. On confirm: delete from Supabase, navigate back to dashboard, refresh list.

### Form submit
Validate: name required, quantity > 0, unit selected, storage location selected. On validation failure: show `showAlert` with specific missing field. Never disable the CTA button â€” show error on tap.

### Pull-to-refresh
`RefreshControl` on dashboard ScrollView, `tintColor={theme.primary}`. Re-fetches all pantry items.

### Low-stock threshold default
If field left empty, store `1` as the threshold value. The executor should not show a placeholder value in the rendered field â€” use `placeholder` text only.

## States Required Per Screen

### PantryDashboard
- Loading: `ActivityIndicator` centered, `theme.primary` color
- Loaded + empty (no items at all): Full empty state UI (see Section Empty State above)
- Loaded + items: grouped list by storage location
- Refreshing: pull-to-refresh indicator

### PantryItemForm
- Add mode: all fields empty (barcode button active)
- Edit mode: all fields pre-filled; delete button visible in header
- Saving: CTA button shows `ActivityIndicator` inline, disabled during network call
- Barcode scanning: camera modal open, scanning state active

### BarcodeScanner modal
- Idle scanning: camera viewfinder live, no indicator
- Scanning / API call: `ActivityIndicator` overlay
- Success: modal closes (no extra state shown in modal)
- Not found: modal closes, toast shown on form screen
