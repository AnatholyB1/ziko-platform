# Phase 1 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none — Tailwind v4 CSS-first only |
| Icon library | none in Phase 1 (footer contains text links only) |
| Font | Inter via `next/font/google` self-hosted from Vercel CDN |

**Rationale:** shadcn uses Tailwind v3-style config and is incompatible with the Tailwind v4 CSS-first approach locked in D-05. No component library is needed in this scaffold phase — the only visible UI element is a footer with text links.

**Font note (source: CONTEXT.md discretion):** Use `next/font/google` with `display: 'swap'` and `subsets: ['latin']`. This self-hosts from Vercel's CDN — no outbound request to Google Fonts (CNIL-compliant, satisfies FOUND-05).

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing |
| md | 16px | Default element padding, footer link gaps |
| lg | 24px | Footer vertical padding |
| xl | 32px | Page-level horizontal gutters |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level top/bottom spacing |

Exceptions: none

**Note (source: CONTEXT.md D-03):** No spacing scale tokens are declared in `@theme` in Phase 1 — these values guide implementation only. The Tailwind v4 default spacing scale is sufficient for the scaffold; custom spacing tokens are added in Phase 3 if needed.

---

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 (regular) | 1.5 |
| Label | 14px | 400 (regular) | 1.5 |
| Heading | 20px | 600 (semibold) | 1.2 |
| Display | 28px | 600 (semibold) | 1.2 |

**Note (source: CONTEXT.md D-03):** No `fontSize` tokens are declared in `@theme` in Phase 1. These values define the base contract for placeholder page headings and footer labels. Tailwind v4 default type scale maps directly: `text-sm` = 14px, `text-base` = 16px, `text-xl` = 20px, `text-2xl` = 28px. Use those utility classes directly — do not add custom font-size tokens in Phase 1.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#F7F6F3` | Page background (`background`) |
| Secondary (30%) | `#FFFFFF` | Footer surface, card surfaces |
| Accent (10%) | `#FF5C1A` | Footer accent border-top only in Phase 1 |
| Text primary | `#1C1A17` | All body copy, footer link text |
| Text muted | `#6B6963` | Footer copyright line, placeholder body copy |
| Border | `#E2E0DA` | Footer divider line, page-level separators |
| Destructive | not applicable | No destructive actions in Phase 1 |

Accent reserved for: footer top border (a 2px `#FF5C1A` rule at the top of the `<footer>` element). No other elements use accent in Phase 1. CTA buttons, interactive states, and hover treatments are Phase 3 concerns.

**Source:** CONTEXT.md D-03, CLAUDE.md Design System table. All five token values are locked — do not alter.

**Tailwind v4 `@theme` declaration (exact — source: FOUND-03):**
```css
@theme {
  --color-primary: #FF5C1A;
  --color-background: #F7F6F3;
  --color-text: #1C1A17;
  --color-border: #E2E0DA;
  --color-muted: #6B6963;
}
```

## Component Inventory

Phase 1 delivers exactly these UI elements:

| Component | Description | Phase 1 scope |
|-----------|-------------|---------------|
| `RootLayout` | `html` + `body` with Inter font class, bg-background color | Scaffold only |
| `LocaleLayout` | Wraps all `[locale]/*` pages — provides i18n context | Scaffold only |
| `Footer` | `<footer>` with 3 legal links + copyright. Fixed to bottom of page. | Visible on every page (FOUND-07) |
| `HomePage` | `<main>` with placeholder heading + one-line body copy | Bilingual placeholder only |
| Legal stub pages | `mentions-legales`, `politique-de-confidentialite`, `cgu` — each renders an `<h1>` with the page title and a body note "Contenu à venir." | Stub only |

No interactive components, no form elements, no buttons, no icons. Footer links are standard `<a>` (or `<Link>`) elements with no hover accent treatment in Phase 1.
