**Function 1: getBalance(userId: string): Promise<{ balance: number }>**
- Upsert into `user_ai_credits` with `{ user_id: userId, balance: 0 }`, onConflict `user_id`, ignoreDuplicates true
- Then select balance. If the upsert-with-select doesn't return the existing row on conflict, do a separate `.from('user_ai_credits').select('balance').eq('user_id', userId).single()` as fallback.
- Return `{ balance: data.balance }` or `{ balance: 0 }` on error.

**Function 2: earnCredits(userId: string, source: string, idempotencyKey: string): Promise<{ credited: boolean }>**
- Per EARN-07: Count today's earn transactions: `.from('ai_credit_transactions').select('*', { count: 'exact', head: true }).eq('user_id', userId).eq('type', 'earn').gte('created_at', todayUTC + 'T00:00:00Z')`.
- If `count >= DAILY_EARN_CAP`, return `{ credited: false }` (cap reached).
- Per EARN-10: Insert with idempotency: `.from('ai_credit_transactions').insert({ user_id: userId, type: 'earn', amount: EARN_AMOUNT, source, idempotency_key: idempotencyKey })`. The partial unique index `(user_id, source, idempotency_key)` enforces ON CONFLICT DO NOTHING at DB level.
- Note: Supabase JS v2 does not have `.onConflict().ignoreDuplicates()` for insert — use `.upsert()` with `onConflict: 'user_id,source,idempotency_key'` and `ignoreDuplicates: true`, OR catch the 409/23505 unique violation error and treat it as a no-op. The safest approach: just `.insert()` and if the error code is `23505`, return `{ credited: false }`.
- If insert succeeds, increment balance: `.from('user_ai_credits').update({ balance: /* raw SQL */ }).eq('user_id', userId)`. Since Supabase JS v2 doesn't support raw SQL in update, use the RPC pattern or a direct `.rpc()` call. The simplest: read current balance, compute new, update. BUT for safety against concurrent earns, use a raw SQL approach. Pragmatic solution: call `.rpc('increment_balance', { p_user_id: userId, p_amount: EARN_AMOUNT })`. BUT that RPC doesn't exist yet.
- **Actual approach for earn balance increment**: Since earn races only ever add (never subtract), a relative UPDATE is safe. Use Supabase's PostgREST raw column update: the Supabase JS client does NOT support `balance = balance + 1` in `.update()`. Instead, read balance, add EARN_AMOUNT, write back. Accept the rare race (two concurrent earns might both read 5, both write 6 instead of 7 — but this is a minor underpayment that benefits the platform, not a correctness issue). OR better: use a one-line SQL RPC. Since we don't want to add a new migration, the pragmatic approach is to just do a read-then-write for earn (the risk is minimal and favors the platform).
- Return `{ credited: true }` on success, `{ credited: false }` on duplicate or cap.

**Function 3: deductCredits(userId: string, action: CreditAction, idempotencyKey: string): Promise<{ success: boolean; balance: number; required?: number }>**
- Call `supabase.rpc('deduct_ai_credits', { p_user_id: userId, p_cost: CREDIT_COSTS[action], p_action_type: action, p_idempotency_key: idempotencyKey })`.
- Parse JSONB response. Return `{ success: data.success, balance: data.balance_after ?? data.balance ?? 0, required: data.required }`.

**Function 4: getQuotaStatus(userId: string, action: CreditAction): Promise<QuotaStatus>**

Define QuotaStatus type:
```typescript
export interface QuotaStatus {
  withinFreeQuota: boolean;
  dailyUsed: number;
  dailyQuota: number;
  balance: number;
  earnHint: string;
}
```

Implementation:
- Compute `todayUTC` as `new Date().toISOString().split('T')[0]`.
- For daily actions (chat, scan): count today's transactions for this action: `.from('ai_credit_transactions').select('*', { count: 'exact', head: true }).eq('user_id', userId).eq('source', action).gte('created_at', todayUTC + 'T00:00:00Z')`.
- For monthly actions (program): compute first of month as `new Date(Date.UTC(new Date().getUTCFullYear(), new Date().getUTCMonth(), 1)).toISOString()`. Count this month's transactions: `.eq('source', 'program').gte('created_at', firstOfMonth)`.
- Count today's earn transactions to determine earned bonus: `.eq('type', 'earn').gte('created_at', todayUTC + 'T00:00:00Z')`. The earned bonus count is `Math.min(earnCount, totalBonusForAction)`.
- For daily: `dailyQuota = DAILY_QUOTAS[action].base + Math.min(todayEarnCount, DAILY_QUOTAS[action].bonus)`. `withinFreeQuota = dailyUsed < dailyQuota`.
- For monthly (program): `monthlyQuota = MONTHLY_QUOTAS.program.base + Math.min(todayEarnCount, MONTHLY_QUOTAS.program.bonus)`. Use same pattern.
- Get balance via `getBalance(userId)`.
- `earnHint`: static rotation for Phase 18 — check which hint to show. Use a simple array: `['Log a workout to earn credits', 'Complete your habits to earn credits', 'Log a meal to earn credits', 'Log a stretch to earn credits', 'Log a cardio session to earn credits', 'Log body measurements to earn credits']`. Pick the first one (Phase 20 will make this dynamic based on today's actual earn state).
- Return the QuotaStatus object.
