**creditCheck(action: CreditAction)** — factory function returning middleware:
1. Get `userId` from `c.get('auth').userId` (authMiddleware has already run).
2. **PREM-02**: Query `user_profiles.tier`: `supabase.from('user_profiles').select('tier').eq('user_id', userId).single()`. If `profile?.tier === 'premium'`, set `c.set('creditPassThrough', true)` and call `return next()`.
3. Call `creditService.getQuotaStatus(userId, action)`.
4. If `quota.withinFreeQuota === true`, set `c.set('creditPassThrough', true)` and call `return next()` — this is the first-N-free pass-through per D-01/D-02.
5. If outside free quota, check balance: `const cost = CREDIT_COSTS[action]`. If `quota.balance < cost`, return the rich 402 per D-04:
   ```typescript
   return c.json({
     error: 'insufficient_credits',
     balance: quota.balance,
     required: cost,
     daily_used: quota.dailyUsed,
     daily_quota: quota.dailyQuota,
     earn_hint: quota.earnHint,
   }, 402);
   ```
6. Otherwise (has balance, outside free quota), set `c.set('creditPassThrough', false)` and call `return next()`.

**creditDeduct(action: CreditAction)** — factory function returning middleware:
1. Call `await next()` first — the handler runs and the response is populated.
2. Read `c.get('creditPassThrough')`. If `true`, return immediately (free quota slot used, no deduction needed).
3. If `c.res.status >= 400`, return immediately — handler failed, do not charge per SC5.
4. Get `userId` from `c.get('auth').userId`.
5. Generate idempotency key: `const idempotencyKey = c.req.header('X-Request-Id') ?? crypto.randomUUID()`.
6. Call `creditService.deductCredits(userId, action, idempotencyKey)` wrapped in `.catch((err) => { console.error('[creditDeduct] deduction failed:', err); })` — fire-and-forget, do not fail the response.

**Important ordering note in comments:**
- `creditCheck` MUST run AFTER `authMiddleware` (reads `c.get('auth')`)
- `creditCheck` MUST run BEFORE the handler (gates access)
- `creditDeduct` MUST run such that `await next()` executes the handler first, then inspects `c.res.status`

Add a comment about streaming routes: for `/ai/chat/stream`, HTTP status is 200 before the stream completes. Errors inside the stream are SSE events, not HTTP status changes. Credit is deducted on the 200 status — this matches Anthropic's billing model (charges per token even on partial responses).
