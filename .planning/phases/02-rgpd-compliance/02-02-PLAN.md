2. Create `src/components/account/DeleteAccountForm.tsx` as a Client Component (`'use client'` at top):

   Imports:
   - `useActionState` and `useState` from `'react'` (NOT from `'react-dom'` — Pitfall 5 in research)
   - `deleteAccount` and `DeleteAccountState` from `'@/actions/account'`

   Component logic:
   - `const initialState: DeleteAccountState = { status: 'idle', message: '' };`
   - `const [state, formAction, pending] = useActionState(deleteAccount, initialState);`
   - `const [typed, setTyped] = useState('');`
   - `const [checked, setChecked] = useState(false);`
   - `const canSubmit = typed === 'SUPPRIMER' && checked && !pending;`

   Rendering — if `state.status === 'success'`:
   - Show a green success box: `<div className="rounded-lg bg-green-50 border border-green-200 p-6"><p className="text-green-800 font-medium">{state.message}</p></div>`
   - Do NOT render the form when in success state.

   Rendering — form (when not success):
   - `<form action={formAction} className="space-y-6 max-w-md">`
   - Email input: `<input type="email" name="email" required placeholder="votre@email.com" className="w-full rounded-lg border border-border px-4 py-3 text-text placeholder:text-muted focus:outline-none focus:ring-2 focus:ring-primary" />`
   - Checkbox label: `<label className="flex items-start gap-3 cursor-pointer"><input type="checkbox" checked={checked} onChange={e => setChecked(e.target.checked)} className="mt-1 h-4 w-4 rounded border-border text-primary focus:ring-primary" /><span className="text-sm text-text">Je comprends que cette action est irr\u00e9versible</span></label>`
   - Hidden field for confirmation: `<input type="hidden" name="confirmation" value={typed} />`
   - Text confirmation: a label "Tapez SUPPRIMER pour confirmer" with an input: `<input type="text" value={typed} onChange={e => setTyped(e.target.value)} placeholder="SUPPRIMER" className="w-full rounded-lg border border-border px-4 py-3 text-text placeholder:text-muted focus:outline-none focus:ring-2 focus:ring-primary" />`
   - Error display: if `state.status === 'error'`, show `<p className="text-red-600 text-sm">{state.message}</p>`
   - Submit button: `<button type="submit" disabled={!canSubmit} className="w-full rounded-lg bg-red-600 px-6 py-3 text-white font-medium disabled:opacity-40 disabled:cursor-not-allowed hover:bg-red-700 transition-colors">{pending ? 'Suppression en cours...' : 'Supprimer mon compte'}</button>`

   Use Ziko design tokens for inputs (border-border, text-text, text-muted, ring-primary). Use red for the delete button (bg-red-600) to signal destructive action.

3. Create `src/app/[locale]/supprimer-mon-compte/page.tsx` as a Server Component:

   Follow the exact pattern from the existing legal page stubs:
   ```typescript
   import { setRequestLocale } from 'next-intl/server';
   import { DeleteAccountForm } from '@/components/account/DeleteAccountForm';

   type Props = { params: Promise<{ locale: string }> };

   export default async function SupprimerMonComptePage({ params }: Props) {
     const { locale } = await params;
     setRequestLocale(locale);

     return (
       <main className="max-w-screen-xl mx-auto px-8 py-16">
         <h1 className="text-2xl font-semibold mb-2">Supprimer mon compte</h1>
         <p className="text-muted mb-8">
           Conformement au RGPD (Art. 17), vous pouvez demander la suppression de votre compte et de toutes vos donnees personnelles.
         </p>
         <DeleteAccountForm />
       </main>
     );
   }
   ```
   Use proper French accented characters (e with accent grave/aigu, etc.) in the paragraph text.

   This page is static — no `getTranslations` needed since the heading/description are hardcoded French (consistent with D-07 approach for French content). The form itself is a Client Component island.
