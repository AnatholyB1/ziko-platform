**1. Create shared helper `apps/mobile/src/lib/earnCredits.ts`:**

This is a new file. It provides a single function used by all 6 mobile screens.

```typescript
import { createClient } from '@supabase/supabase-js';

const API_URL = process.env.EXPO_PUBLIC_API_URL ?? '';

/**
 * Fire-and-forget credit earn call to POST /credits/earn.
 * MUST NOT throw — activity save must succeed regardless of earn outcome (SC-3, D-10).
 *
 * @param supabase - Supabase client instance (for reading auth session)
 * @param source - Activity source: 'workout' | 'habit' | 'meal' | 'measurement' | 'stretch' | 'cardio'
 * @param idempotencyKey - Unique key preventing double-credit (record UUID or deterministic string)
 */
export async function callCreditsEarn(
  supabase: { auth: { getSession: () => Promise<{ data: { session: { access_token: string } | null } }> } },
  source: string,
  idempotencyKey: string,
): Promise<void> {
  try {
    const { data: { session } } = await supabase.auth.getSession();
    if (!session?.access_token) return;

    fetch(`${API_URL}/credits/earn`, {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
        'Authorization': `Bearer ${session.access_token}`,
      },
      body: JSON.stringify({ source, idempotency_key: idempotencyKey }),
    }).catch(() => {}); // Fire-and-forget: do not await fetch
  } catch {
    // Swallow all errors — earn must never block activity save
  }
}
```

Key design: The outer try/catch handles getSession failures. The inner `.catch(() => {})` on fetch handles network errors. The fetch itself is NOT awaited — truly fire-and-forget.

| Boundary | Description |
|----------|-------------|
| mobile app -> POST /credits/earn | Untrusted client sends source + key over HTTPS |
| supabase.auth.getSession -> token | Session token read from secure storage on device |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-20-06 | Spoofing | callCreditsEarn Authorization header | mitigate | Token read from supabase.auth.getSession() (secure storage); backend authMiddleware validates JWT server-side |
| T-20-07 | Information Disclosure | API_URL in client bundle | accept | API URL is public (same as all Expo apps); no secret leaked; auth required for all endpoints |
| T-20-08 | Tampering | Mobile sends arbitrary source/key | mitigate | Server-side VALID_SOURCES allowlist (Plan 01); idempotency key enforced by partial unique index; daily cap limits total earn |
| T-20-09 | Denial of Service | Rapid fire-and-forget calls flooding endpoint | mitigate | Phase 12 rate limiter (Upstash Redis) applies per-user rate limit to all API routes including /credits/earn |
</threat_model>
