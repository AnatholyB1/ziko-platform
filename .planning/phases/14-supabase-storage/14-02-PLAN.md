Create the `GET /storage/upload-url` backend route that returns Supabase signed upload URLs for authorized clients.

Purpose: Mobile clients must obtain a signed URL before uploading directly to Supabase Storage — this is the only way to bypass Vercel's 4.5 MB body limit. This route validates bucket allowlist and path-prefix ownership, then delegates to the Supabase SDK.

Output: New `backend/api/src/routes/storage.ts` and updated `backend/api/src/app.ts` mounting the new router.

**Step 5 — GET /upload-url handler (per D-10, D-11, D-12, D-13, D-14, D-15):**
```typescript
storageRouter.get('/upload-url', async (c) => {
  const bucket = c.req.query('bucket');
  const path = c.req.query('path');
  const { userId } = c.get('auth');

  // Validate required params
  if (!bucket || !path) {
    return c.json({ error: 'bucket and path query params are required' }, 400);
  }

  // Validate bucket allowlist (D-12)
  if (!ALLOWED_BUCKETS.includes(bucket as AllowedBucket)) {
    return c.json({ error: `Invalid bucket. Allowed: ${ALLOWED_BUCKETS.join(', ')}` }, 400);
  }

  // Validate path prefix ownership (D-13)
  if (!path.startsWith(`${userId}/`)) {
    return c.json({ error: 'Path must start with your user ID' }, 403);
  }

  // Generate signed upload URL (D-14)
  const { data, error } = await supabase.storage
    .from(bucket)
    .createSignedUploadUrl(path, { expiresIn: 60 });

  if (error || !data) {
    console.error('[Storage] createSignedUploadUrl error:', error);
    return c.json({ error: 'Failed to generate upload URL' }, 500);
  }

  // Return upload_url, path, token (D-15)
  return c.json({
    upload_url: data.signedUrl,
    path: data.path,
    token: data.token,
  });
});
```
