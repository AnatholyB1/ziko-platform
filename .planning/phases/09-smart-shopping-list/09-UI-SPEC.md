## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing, tab bar paddingTop |
| md | 16px | Default element spacing, card padding, list row padding |
| lg | 24px | Section padding, horizontal scroll padding |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

**Exceptions:**
- Tab bar height: 56px + `insets.bottom` (matches existing PantryTabBar pattern exactly)
- Tab bar icon size: 22px (matches existing PantryTabBar)
- Tab bar label font size: 11px (matches existing PantryTabBar)
- ScrollView `paddingBottom`: 100px (all plugin screens — per CLAUDE.md)
- Checkbox touch target: 44px minimum (accessibility — wrap in 44×44 touchable area)
- Row `borderRadius`: 14px (matches PantryItemRow pattern)

**Source:** PantryDashboard.tsx measurements + CLAUDE.md.

## Interaction Contracts

### Check-off flow (SHOP-03)
1. User taps a `ShoppingListItemRow`
2. Optimistic: row disappears immediately from list
3. Background: Supabase `DELETE` on `shopping_list_items` where `id = item.id`
4. If `pantry_item_id` is set: Supabase `UPDATE pantry_items SET quantity = low_stock_threshold ?? 1`
5. On Supabase error: row re-appears at original position, `showAlert('Erreur', 'Impossible de mettre à jour. Réessayez.')` — use `showAlert` from `@ziko/plugin-sdk`, never `Alert` from `react-native`

### Add from recipe flow (SHOP-02, D-04, D-05)
1. User taps "Ajouter à la liste" on RecipeDetail
2. Filter `recipe.ingredients` to those with no `pantry_item_id` match in user's pantry items (cross-reference by name, case-insensitive)
3. For each missing ingredient: check if already in `shopping_list_items` by name (case-insensitive). Skip if found (D-05).
4. Insert remaining into `shopping_list_items` with `source: 'recipe'`, `recipe_name: recipe.name`
5. On success: no modal — silent (button stays as-is)
6. On error: `showAlert('Erreur', "Impossible d'ajouter à la liste. Réessayez.")`

### Export flow (SHOP-04)
1. User taps share icon in header
2. Build plain text: header `"Liste de courses Ziko\n\n"` + alphabetically sorted flat list, one item per line
3. Format per item: `- {name} × {quantity} {unit}` (if quantity known) or `- {name}` (if quantity/unit null)
4. Call `Share.share({ message: text })` from `react-native`
5. No error handling required for share sheet (native OS handles dismissal)

### Loading state
- Initial load: `ActivityIndicator` size `"large"` color `theme.primary`, centered on screen
- Refresh: `RefreshControl` tintColor `theme.primary` on `ScrollView`
- No skeleton — matches existing pantry pattern
