**Step A — Deploy to production:**

Option 1 (CLI — if Vercel CLI is installed):
```bash
cd /c/ziko-web && vercel --prod --yes
```

Option 2 (Git push — if Vercel is connected to the repo):
Push the latest commits to the main branch. Vercel will auto-deploy to production.

Option 3 (Dashboard):
Go to Vercel Dashboard -> your project -> Deployments -> trigger a production deploy.

Wait for the deploy to complete successfully.

**Step B — Add custom domain (D-02):**
1. Go to Vercel Dashboard -> your ziko-web project -> Settings -> Domains
2. Add domain: `ziko-app.com`
3. Vercel will show DNS configuration instructions. Follow them at your domain registrar:
   - **Option A (recommended):** Set nameservers to Vercel's nameservers (ns1.vercel-dns.com, ns2.vercel-dns.com)
   - **Option B:** Add a CNAME record pointing `ziko-app.com` to `cname.vercel-dns.com`
   - Also add `www.ziko-app.com` as a redirect domain if desired
4. Wait for DNS propagation (typically 5-30 minutes, can take up to 48 hours)
5. Vercel will auto-provision the SSL/TLS certificate once DNS resolves

**Step C — Verify HTTPS is live:**
Visit `https://ziko-app.com` in a browser. Confirm:
- The page loads with no certificate warnings
- The Ziko marketing site content appears (Hero with real screenshot, not placeholder)
- The URL bar shows a lock icon (valid HTTPS)
