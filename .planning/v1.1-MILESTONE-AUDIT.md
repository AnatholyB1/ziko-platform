## Tech Debt Summary

### Phase 09 — Smart Shopping List

**SHOP-03 recipe check-off does not create pantry item** (UX gap)

Recipe-sourced shopping list items checked off do not trigger a `pantry_items` insert. A user who buys an ingredient that wasn't in their pantry at all will not see it appear in their pantry after checking it off. The item is simply removed from the shopping list.

*Mitigation path:* `handleCheckOffRecipe()` could prompt "How much did you buy?" and then insert to `pantry_items`. Deferred to v1.2.

**SHOP-03 pantry quantity restore is threshold+1** (UX gap)

Low-stock check-off restores to `threshold + 1` — the minimum to clear the low-stock badge. Not a user-specified restock quantity. Functionally correct (badge disappears, item re-appears next purchase cycle), but imprecise.

*Mitigation path:* Same "how much did you buy?" prompt. Deferred to v1.2.

### Phase 07 — AI Recipe Suggestions

**Missing VALIDATION.md** — Nyquist coverage absent. Phase is verified and passes, but no validation contract was written.

### Phase 06, 08 — Stale VALIDATION.md files

Pre-execution draft VALIDATION.md files were never updated post-execution. All Wave 0 checks would pass (artifacts exist), but the files still read as "PENDING". Can be updated or treated as historical documentation.

### Phase 08 — pantry_log_recipe_cooked AI tool not built

Plan 08-01 mentioned building a `pantry_log_recipe_cooked` AI tool, but the feature was implemented via direct Supabase calls in `RecipeConfirm.tsx` instead. The AI tool was never built. This is functionally equivalent and arguably better (simpler architecture), but CLAUDE.md tool registry documentation does not reflect this.
